61. JS Errors

JavaScript memiliki berbagai jenis error yang dapat terjadi selama eksekusi:

  • SyntaxError: Kesalahan saat menulis sintaks yang tidak valid, misalnya kurung yang hilang.
  • ReferenceError: Terjadi ketika mencoba mengakses variabel yang tidak didefinisikan.
  • TypeError: Terjadi jika operasi dilakukan pada tipe data yang salah, misalnya memanggil properti dari null.

Error dapat ditangani menggunakan blok try...catch:

try {
    let x = y; // ReferenceError
} catch (error) {
    console.log("Error:", error.message);
}

62. JS Scope

Scope menentukan area di mana variabel dapat diakses. Ada tiga jenis utama scope:

  • Global Scope: Variabel yang dideklarasikan di luar fungsi atau blok dapat diakses dari mana saja.
  • Function Scope: Variabel yang dideklarasikan dalam fungsi hanya dapat diakses di dalam fungsi tersebut.
  • Block Scope: Dengan let atau const, variabel hanya dapat diakses di dalam bloknya.
let a = 10; // Global scope
function test() {
    let b = 20; // Function scope
    if (true) {
        let c = 30; // Block scope
    }
    console.log(b); // OK
    console.log(c); // Error
}

63. JS Hoisting

Hoisting memungkinkan variabel dan fungsi digunakan sebelum dideklarasikan:

  • Hanya berlaku untuk var, fungsi, dan deklarasi class.
  • Variabel dengan let dan const tidak dapat digunakan sebelum deklarasi.
console.log(a); // undefined (hoisted)
var a = 5;

console.log(b); // Error (let tidak di-hoist)
let b = 10;

64. JS Strict Mode

Strict mode digunakan untuk menegakkan aturan yang lebih ketat dalam JavaScript:

  • Mencegah penggunaan variabel tanpa deklarasi.
  • Menghindari kesalahan umum, seperti duplikasi parameter di fungsi.
"use strict";
x = 10; // Error: x tidak dideklarasikan

65. JS this Keyword

this merujuk pada objek yang menjalankan fungsi tertentu:

  • Dalam fungsi global, this merujuk ke window.
  • Dalam fungsi di objek, this merujuk ke objek tersebut.
const obj = {
    name: "John",
    greet: function() {
        return "Hello, " + this.name;
    }
};
console.log(obj.greet());

66. JS Arrow Function

Arrow function adalah cara singkat untuk menulis fungsi:

  • Tidak memiliki this, sehingga cocok untuk callback.
  • Menawarkan sintaks yang lebih sederhana.
const add = (a, b) => a + b;

67. JS Classes

Kelas adalah cara untuk membuat blueprint dari objek:

class Animal {
    constructor(name) {
        this.name = name;
    }
    speak() {
        console.log(this.name + " makes a sound");
    }
}
const dog = new Animal("Dog");
dog.speak();

68. JS Modules

Modul memungkinkan pengelolaan kode menjadi file terpisah:

// File: module1.js
export const greet = () => "Hello";

// File: main.js
import { greet } from './module1.js';
console.log(greet());

69. JS JSON

JSON (JavaScript Object Notation) adalah format yang digunakan untuk pertukaran data:

const jsonData = '{"name": "John", "age": 30}';
const obj = JSON.parse(jsonData); // Mengubah JSON ke objek
console.log(obj.name);

70. JS Debugging

Debugging membantu menemukan dan memperbaiki kesalahan dalam kode:

  • Gunakan console.log() untuk memeriksa nilai variabel.
  • Gunakan debugger untuk menghentikan eksekusi di titik tertentu.

71. JS Style Guide

  • Gunakan camelCase untuk penamaan variabel.
  • Hindari penggunaan var, gunakan let atau const.

72. JS Best Practices

Praktik terbaik dalam menulis kode JavaScript untuk meminimalkan bug dan meningkatkan efisiensi:

  • Selalu gunakan === untuk perbandingan tipe dan nilai.
  • Gunakan const untuk variabel yang tidak berubah dan let untuk variabel yang dapat berubah.
  • Hindari penggunaan variabel global sebanyak mungkin.
  • Berikan nama variabel dan fungsi yang jelas dan deskriptif.

73. JS Mistakes

Kesalahan umum yang sering dilakukan saat menulis JavaScript:

  • Melupakan deklarasi variabel menggunakan let, const, atau var.
  • Menggunakan variabel global tanpa sengaja, yang dapat menyebabkan konflik.
  • Tidak memeriksa jenis data sebelum melakukan operasi, misalnya menambahkan string dan angka.
  • Tidak menangani error menggunakan try...catch.

74. JS Performance

Tips untuk meningkatkan performa kode JavaScript:

  • Kurangi manipulasi DOM karena itu memerlukan waktu proses yang besar.
  • Gunakan let dan const untuk menghindari kebocoran memori.
  • Cache data yang sering digunakan untuk mengurangi permintaan ulang.
  • Hapus event listener yang tidak diperlukan untuk mencegah kebocoran memori.

75. JS Reserved Words

Kata-kata yang dicadangkan dalam JavaScript tidak dapat digunakan sebagai nama variabel atau identifier:

  • break, case, class, const, continue
  • debugger, default, delete, do, else
  • export, extends, finally, for, function
  • if, import, let, return, switch
  • throw, try, var, while, yield
